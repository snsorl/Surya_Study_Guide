# Exercise: Evaluating AI-Generated Edge-Case Tests

## Learning Objectives
- Formulate effective prompts to request unit tests from AI assistants.
- Evaluate AI-generated code snippets for syntax accuracy and logical correctness.
- Identify valid vs. invalid edge-case testing conditions.
- Document test evaluation parameters in structured formats.

---

## Setup Instructions
1. Ensure your `StringCalculator` class from the previous lab is fully implemented.
2. Open your AI coding assistant panel (such as Gemini Code Assist) within your IDE workspace.
3. Prepare a test log template to document your evaluation findings.

---

## Step-by-Step Tasks

### Task 1: Prompt the AI for Edge-Case Tests
Write and send a structured prompt to your AI assistant. You must demand edge-case coverage for your StringCalculator methods.

#### Recommended Prompt Template:
> Act as a Senior Quality Assurance Engineer. Review the following Java utility requirements:
> A StringCalculator class has three methods: add(String), multiply(String), and divide(String).
> Each method parses a comma-separated string of integer numbers.
> Generate a JUnit 5 test class covering at least 5 complex edge cases for these methods (e.g. empty strings, null inputs, negative values, trailing commas, spaces, or division boundaries).
> Return ONLY the JUnit 5 Java code block. Do not write any conversational text or explanations.

---

### Task 2: Audit and Evaluate the AI's Output
Review the code block generated by the AI assistant. Do not copy-paste it directly. Analyze the logic carefully to answer the following questions:

1.  **Compilation Check:** Does the code use valid JUnit 5 annotations and standard assertions?
2.  **Logical Veracity:** Are the test assertions correct based on the calculator's constraints? (e.g., if the AI asserts that `add(" , ")` returns `0`, does your calculator actually handle that without throwing a `NumberFormatException`?)
3.  **Redundancy Audit:** Did the AI generate duplicate tests (e.g., testing empty string `""` and whitespace `" "` using identical logic)?

---

### Task 3: Document and Remediate
Create a markdown table in your local notebook summarizing your findings for each test the AI suggested:

| Test Case Suggested | Intent/Edge Case Covered | Evaluation (Valid / Invalid / Redundant) | Action Required |
|---|---|---|---|
| `testAddWithNull()` | Null input safety. | Valid. | Implement null check if missing in source. |
| `testAddWithSpace()`| Handles padded spaces in tokens. | Valid. | Add `.trim()` before parsing tokens. |
| ... | ... | ... | ... |

*Remediation:* If any valid edge cases suggested by the AI are not handled by your current `StringCalculator.java` implementation, update your code to handle them and ensure all tests pass.

---

## Definition of Done
-   A documented prompt log evaluation matrix containing at least 5 audited test cases.
-   Updated `StringCalculator.java` that handles at least 2 new edge cases suggested by the AI.
-   All JUnit tests compile and pass successfully in the IntelliJ runner.
