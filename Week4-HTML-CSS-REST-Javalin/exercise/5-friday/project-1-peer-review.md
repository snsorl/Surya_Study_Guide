# Exercise: Project 1 Peer Review

## Objective
Evaluate the final project presentations of at least two other developer groups. Provide objective, detailed technical feedback on their database schema structures, REST API architectures, front-end usability, and security configurations.

---

## Scenario
In professional development squads, giving and receiving peer feedback is a core component of engineering growth. Conducting code audits and architecture reviews on other projects helps you spot issues in your own designs and improves your technical communication.

---

## Core Tasks

### 1. Presentation Audits
- Attend the live Project 1 demonstrations.
- Choose two presenting groups to evaluate.
- During their presentation, inspect their running system and take notes on their database relationships, API routing endpoints, frontend styled layouts, and security handling.
- Reference: [project-1-presentations.md](file:///c:/Learning/INF-JFSD/content/Week4-HTML-CSS-REST-Javalin/written/5-friday/project-1-presentations.md)

### 2. Complete the Evaluation Matrix
For each of the two selected groups, construct a feedback review containing these evaluation criteria:
-   **Database Design**: Are primary and foreign key constraints set up correctly? Is the schema normalized?
-   **API Routing & REST Quality**: Did they map resource endpoints to semantic HTTP verbs and return correct status codes?
-   **Frontend Implementation**: Is the layout styled clean with CSS, responsive, and easy to navigate?
-   **Security Configuration**: Did they map exception boundaries globally? Is there structured logging?
-   **Critical Insight**: Document at least one positive technical highlight and one constructive architectural recommendation for improvement.

### 3. Review Submission
- Save your evaluations in a markdown file named `peer-reviews.md` inside your personal workspace folder.

---

## Definition of Done
Your submitted peer review document must contain:
- A detailed review of at least two presenting groups.
- Explicit scoring for database, routing, styling, and security sections.
- Constructive feedback points referencing topics taught during the week (e.g. CORS setup, Jackson DTO mapping, Box Model alignments, exception mapping).
