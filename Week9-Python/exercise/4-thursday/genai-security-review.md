# Exercise: Security Review of AI-Generated Code

## Objectives
- Conduct a security code review on AI-generated authentication scripts.
- Identify common security flaws: SQL injection, hardcoded credentials, and weak hashing algorithms.
- Refactor insecure code patterns into secure, production-ready backend code.

---

## The Scenario
Your team asked a Generative AI tool to generate a Flask authentication login route that checks user credentials against a SQLite database. The model generated working code, but you suspect it contains critical vulnerabilities. You must analyze the code, identify the security defects, and write a secured replacement.

---

## The Vulnerable Code Snippet
Below is the code generated by the AI. Review it carefully:

```python
import sqlite3
import hashlib
from flask import Flask, request, jsonify

app = Flask(__name__)

# Hardcoded Admin credentials for debugging fallback
ADMIN_USER = "system_admin"
ADMIN_KEY = "SuperSecurePassword123"

@app.route("/login", methods=["POST"])
def login():
    username = request.json.get("username")
    password = request.json.get("password")
    
    # Check Admin fallback
    if username == ADMIN_USER and password == ADMIN_KEY:
        return jsonify({"message": "Admin login successful", "token": "STATIC-TOKEN-XYZ"}), 200
        
    # Standard user lookup in SQLite database
    # Hashing the incoming password using MD5
    hashed_password = hashlib.md5(password.encode()).hexdigest()
    
    conn = sqlite3.connect("users.db")
    cursor = conn.cursor()
    
    # Query database for matching user
    query = f"SELECT * FROM users WHERE username = '{username}' AND password = '{hashed_password}'"
    cursor.execute(query)
    user = cursor.fetchone()
    
    conn.close()
    
    if user:
        return jsonify({"message": "Login successful"}), 200
    else:
        return jsonify({"error": "Invalid credentials"}), 401
```

## Instructions

### Part 1: Security Audit Report
Create a markdown file named `audit_report.md`. Document every security vulnerability in the AI-generated code. For each issue, identify:
1.  The line number and code segment.
2.  The name of the vulnerability (e.g., SQL Injection, Hardcoded Secrets, Weak Cryptographic Algorithm).
3.  The risk level (High/Medium/Low).
4.  How an attacker could exploit it.

### Part 2: Refactoring
Create a script named `secure_login.py`. Refactor the code to eliminate all vulnerabilities:
- **Secrets Management:** Remove hardcoded credentials. Load them from environmental variables or secure configuration settings.
- **SQL Parametrization:** Avoid string formatting (`f""`) or concatenation inside SQL queries. Use standard parameter placeholders (e.g., `?` or `%s`) to query SQLite:
  ```python
  cursor.execute("SELECT * FROM users WHERE username = ? AND password = ?", (username, hashed_password))
  ```
- **Strong Cryptography:** Replace MD5 with a secure, salted password hashing algorithm (like `hashlib.pbkdf2_hmac` or `bcrypt`).

---

## Definition of Done
Your review lab is complete when:
- `audit_report.md` is generated identifying at least **three distinct vulnerabilities** in the code block.
- `secure_login.py` is written and compiles cleanly, containing refactored, secured logic that fixes all three identified issues.
